# ANALYSIS_MANIFESTO.md

## Purpose

The objective of this project is **not** to predict the future.

The objective is to explain the current market using objective evidence.

This distinction must guide every design decision.

---

## Core Principles

### 1. Data First

Every conclusion must originate from measurable market data.

Never start with an opinion.

Always start with evidence.

---

### 2. Explain Before Predicting

The platform should primarily answer:

> "What is happening?"

before attempting to answer:

> "What might happen next?"

Users trust explanations more than bold predictions.

---

### 3. Transparency Over Certainty

If confidence is low, say so.

If evidence conflicts, show both sides.

Never pretend certainty where none exists.

---

### 4. No Hidden Reasoning

Every market conclusion should be explainable.

Every bullish conclusion.

Every bearish conclusion.

Every neutral conclusion.

Users should always be able to understand why the platform reached its conclusion.

---

### 5. AI Is a Writer

AI is not an analyst.

AI is not a trader.

AI is not a predictor.

AI's only responsibility is transforming validated evidence into clear professional writing.

---

### 6. Every Statement Needs Evidence

Every sentence should be traceable.

Example:

> "The trend remains bullish."

Supporting evidence:

- Price above EMA200
- EMA20 > EMA50 > EMA200
- Higher Highs confirmed
- Higher Lows confirmed

No evidence. No sentence.

---

### 7. Show Bullish And Bearish Evidence

Never write one-sided analysis.

Every report must include:

- **Bullish Factors**
- **Bearish Factors**
- **Risk Factors**

This increases trust.

---

### 8. Avoid Absolute Language

**Never use:**

| Word / Phrase |
|---------------|
| Guaranteed |
| Certain |
| Will definitely |
| 100% |
| Sure thing |
| Risk free |

**Instead use:**

| Word / Phrase |
|---------------|
| Suggests |
| Indicates |
| Supports |
| May |
| Could |
| Likely if confirmed |

---

### 9. Confidence Is Not Probability

The application must not invent probabilities.

Confidence represents the **strength of supporting evidence**.

It does not represent the chance that price will move in a certain direction.

---

### 10. Educational Value

Every report should teach something.

When mentioning an indicator, briefly explain why it matters in the current context.

The platform should make users better at understanding markets.

---

### 11. Institutional Tone

The platform should sound like a professional market research report.

Not a crypto influencer.

Avoid hype.

Avoid emotional language.

Avoid sensationalism.

---

### 12. Continuous Improvement

Historical analyses should be evaluated.

If the engine performs poorly, improve the rules.

Never manipulate statistics.

Never hide mistakes.

Accuracy is earned through iteration.

---

## Final Principle

> The platform earns trust because every conclusion can be verified.
>
> If a conclusion cannot be verified, it should not be published.
